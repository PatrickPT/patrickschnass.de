# Execution Protocol: Strategic AI Consulting Site Build (uv Edition)
# Context: Uses artifacts @PROJECT_MASTER_PLAN.md, @HUGO_CONFIG.md, and @CONTENT_STRATEGY.md

**Role:** You are a Senior Developer specializing in Python (uv) and Hugo.
**Goal:** Execute the build of the website sequentially using a reproducible virtual environment.

---

## STEP 1: Environment & Project Initialization
1. Initialize the project with uv in the current directory:
   `uv init`
2. Add the Python wrapper for Hugo Extended (Crucial for Blowfish/Tailwind):
   `uv add hugo-extended`
3. **Verify Installation:** Run `uv run hugo version` to ensure it works.
4. Initialize the Hugo site structure (Force required due to existing uv files):
   `uv run hugo new site . --force --format toml`
5. **Install Theme:** Initialize git and add Blowfish:
   `git init` (if not already a repo)
   `git submodule add -b main https://github.com/nunocoracao/blowfish.git themes/blowfish`
6. **Cleanup:** Delete the default `hugo.toml` generated by Step 4.

## STEP 2: Configuration (The Brain)
1. Create a NEW `hugo.toml`.
2. Populate it strictly with the content defined in **@HUGO_CONFIG.md**.
3. CRITICAL: Ensure `colorScheme = "blowfish"` and `background = "graph"` are set.
4. Configure the menu to have: "Home", "Services" (anchor to #services), "About", "LinkedIn".

## STEP 3: The "Clean White" Styling (The Look)
1. Create `assets/css/custom.css`.
2. Insert the CSS rules defined in **@HUGO_CONFIG.md** (Section 2) to enforce the white background and canvas opacity.

## STEP 4: Content Injection (The Meat)
1. Create `content/_index.md`.
   - Use the **Profile Layout**.
   - Insert the Headlines from **@CONTENT_STRATEGY.md**.
   - Use `{{< card >}}` shortcodes for the "Power Three" services.
   - Use `{{< chart >}}` shortcode to visualize an 'Expertise Radar' (Strategy, ROI, Tech, Governance).
2. Create `content/about.md`.
   - Insert the "About Me" narrative from **@CONTENT_STRATEGY.md**.

## STEP 5: Legacy & Verification
1. Create `content/posts/archive/_index.md` with `title: "Archive"` and `build: {list: never}`.
2. Create a dummy post `content/posts/archive/legacy-test.md` to ensure the archive works.
3. **Final Verification Command:**
   Run `uv run hugo server -D` and confirm the site builds without errors.

---

**Instruction to AI:**
Execute these steps strictly in order. Always use `uv run hugo` instead of just `hugo`. Confirm completion of each step before moving to the next.
